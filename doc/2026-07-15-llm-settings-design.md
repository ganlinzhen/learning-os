# LLM 设置页设计

## 目标

在 Learning OS 的设置页中，让用户直接配置 DeepSeek 的 API Key、Base URL 与模型名称，并让配置在本机持久保存、对后续导入立即生效。

## 范围

- 新增“设置”导航入口和 DeepSeek LLM 配置页。
- 支持读取配置状态、保存配置和清除已保存的 API Key。
- 保存位置位于应用数据目录，不写入仓库、安装包资源目录或 `apps/generator/.env`。
- 提供“保存并测试连接”操作，验证当前 Base URL、API Key 和模型名称是否可用。
- 首期只支持 DeepSeek 的 API Key、Base URL 和模型名称，不实现多 Provider、模型列表或 Embedding 配置。

## 配置与安全

本地服务在应用根目录的 `settings/llm.json` 原子写入配置，并将文件权限限制为当前用户可读写。API Key 以明文保存在此私有本地文件中，理由是当前应用没有跨平台密钥链抽象；接口绝不返回密钥本身，只返回 `apiKeyConfigured` 状态。

读取接口返回 Base URL、模型名称与密钥是否已配置。保存接口中 API Key 字段是可选的：省略或留空表示保留旧密钥；显式清除动作会删除密钥。Base URL 必须为 HTTP(S) 地址，模型名称必须为非空字符串。

## 数据流

Shell 在桌面版启动 Server 与 Generator 时，把同一个 `LEARNING_OS_LLM_CONFIG_PATH` 传给两个进程；开发脚本也使用该路径。Server 用它实现设置 API，Generator 在每次生成前从该路径读取最新配置，因此设置保存后不需要重启即可对下一次导入生效。Generator 在没有该环境变量时保留既有 `.env` 回退，保障现有独立开发与测试路径。

“保存并测试连接”先原子保存当前配置，再调用 Generator 的本地测试端点。该端点使用已保存配置向 DeepSeek Chat Completions 发起一个最小请求，成功才返回“连接成功”。网络、认证、模型或响应异常均返回明确失败状态；失败配置会保留，方便用户修正后再次测试。

## 界面与交互

左侧导航新增“设置”。页面采用参考图的单列、大留白、分组标题和细边框面板语言，但沿用 Learning OS 的现有色彩与排版。页面只展示“LLM 配置 / DeepSeek”分组：

- API Key 使用密码输入框和“已配置/未配置”状态，永不回填明文。
- Base URL 与模型名称使用右侧输入框和简短说明。
- “保存配置”和“保存并测试连接”并列提供；测试会产生一次真实模型请求。已配置密钥时提供“清除 API Key”并二次确认。
- 保存中禁止重复提交；字段校验和保存失败以就近错误文案呈现；成功后显示短暂成功提示。

## 测试

- Server：配置文件首次读取、原子保存、权限、密钥脱敏、字段校验、清除密钥、测试请求的保存与错误映射、Generator 读取共享路径及 `.env` 回退。
- Generator：最小测试请求的成功、认证失败、网络失败和模型失败映射。
- Console：设置路由和导航、初始加载、掩码状态、保存 payload、测试中的禁用状态、成功/失败提示、清除确认。
- Shell：开发态与打包态均为 Server 和 Generator 注入相同配置路径。
