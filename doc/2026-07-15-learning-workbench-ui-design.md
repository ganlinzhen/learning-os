# Learning OS 成长工作台 UI 重构设计

## 1. 目标与边界

将当前以单页表单为主的控制台，重构为面向个人学习闭环的成长工作台。用户进入应用后应立即知道：今天该做什么、已积累了什么、接下来可以继续什么。

本次改造服务于既有闭环：导入、整理确认、知识库、每日复习与搜索。不新增社交、排行榜、积分商城或复杂数据分析能力。

已确认的设计选择：

- 布局方向：A「今日行动优先」
- 视觉方向：动能蓝紫
- 反向约束：不做复杂数据后台，不做强游戏化打卡应用
- 无障碍范围：不额外增加专项能力，但保持基础可读性、清晰状态与非颜色单一提示

## 2. 信息架构

### 2.1 全局框架

桌面端采用固定侧栏、顶部全局工具栏与内容画布的三层结构。

```text
侧栏：工作台、导入、知识库、复习、搜索
顶部：当前位置、全局搜索（⌘K）、用户菜单
内容：当前页面的主任务与辅助信息
```

侧栏将「工作台」作为新的默认首页，原有导入页移至独立入口。导航只保留图标和文字，不放置统计数字，避免侧栏变成第二个仪表盘。

### 2.2 工作台首页

首页按行动优先级自上而下排列：

1. 今日行动横幅：显示待复习卡片数、预计时间和单一主操作「开始复习」。
2. 学习轨迹：展示本周完成趋势与本周目标进度，用于让用户感知稳定积累，而不是排名比较。
3. 知识库概览：展示已沉淀知识点总数与本周新增量。
4. 快捷操作：导入内容、继续整理。后者在有待确认导入时显示数量。

首页不放大展示多个指标，不增加表格、筛选器或多层钻取。每一个模块都必须提供一个明确的后续动作或状态解释。

### 2.3 原有功能页

- 导入：保留输入与发起整理能力，标题区域增加「导入内容」与简短辅助说明。
- 整理确认：保持知识点审核为主任务，顶部显示来源、候选数量和确认入库操作。
- 知识库：以可扫描的知识点列表为主，补充空状态与最近新增提示。
- 复习：首屏聚焦单张卡片和评分动作，完成后返回带完成反馈的工作台。
- 搜索：保留独立页面作为完整结果页；全局工具栏的搜索框可跳转至该页并带入关键词。

## 3. 视觉系统

### 3.1 色彩

色彩采用克制的动能蓝紫，不使用大面积渐变或多色装饰。

| 角色 | 值 | 使用方式 |
| --- | --- | --- |
| 深墨蓝 | `#131C38` | 侧栏与深色文字 |
| 主蓝 | `#5868F2` | 主操作、当前导航、核心进度 |
| 成长紫 | `#8764F5` | 今日行动横幅的辅助色 |
| 薄荷绿 | `#32BA86` | 完成、正向变化与成功状态 |
| 画布灰白 | `#F7F8FC` | 页面背景 |
| 表面白 | `#FFFFFF` | 面板、输入框和菜单 |
| 边界灰 | `#E4E7EF` | 低强调边界 |
| 次级文字 | `#697386` | 说明、元信息与禁用状态 |

主蓝只用于当前动作、选中状态和关键进度。薄荷绿只用于完成结果，不能作为第二主色扩散到普通组件中。

### 3.2 字体与密度

使用系统无衬线字体栈：`Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "PingFang SC", sans-serif`。移除当前的衬线字体与纸张渐变背景。

- 页面标题：28px / 700
- 模块标题：16px / 700
- 正文：14px / 400
- 标签与辅助信息：12px / 500
- 标准控件高度：40px
- 卡片圆角：12px，输入框与按钮圆角：8px

页面以轻边界组织层次，避免当前「边框 + 大范围柔阴影」同时叠加的卡片样式。仅今日行动横幅使用实色背景建立首屏焦点。

## 4. 组件与交互

### 4.1 核心组件

| 组件 | 用途 | 必要状态 |
| --- | --- | --- |
| 主操作按钮 | 开始复习、开始整理、确认入库 | 默认、悬停、按下、禁用、加载 |
| 次操作按钮 | 导入内容、继续整理 | 默认、悬停、按下、禁用 |
| 导航项 | 在模块间切换 | 默认、悬停、当前页 |
| 今日行动横幅 | 聚焦当前复习任务 | 有任务、无任务、加载 |
| 学习轨迹 | 展示最近七天的学习完成量 | 有数据、无数据 |
| 知识库概览 | 展示累计知识与近期增量 | 有数据、空知识库 |
| 全局搜索 | 快速进入搜索结果 | 默认、输入、加载、无结果 |

### 4.2 反馈原则

- 完成复习后，以简短的完成摘要反馈本次完成量、今日剩余量和本周进度变化。
- 连续学习天数为辅助信息，不作为失败或惩罚机制；中断后不展示挫败文案。
- 加载中使用与内容结构一致的骨架占位，不使用页面中央旋转图标。
- 没有待复习卡片时，横幅改为「今天的复习已完成」，主操作变为「导入内容」或「浏览知识库」。
- 空知识库与无搜索结果需说明下一步，例如「导入一段内容，开始建立知识库」。

### 4.3 动效

仅用于操作结果和状态变化：按钮按下、导航切换、进度变化、复习完成。常规过渡控制在 150–200ms；不做页面入场编排、弹跳或连续刷新的图表动画。

## 5. 数据与实现边界

当前已有接口可直接支持待复习数、知识点列表与搜索结果。工作台还需要聚合以下信息：

- 今日待复习卡片数量与完成数量。
- 已入库知识点数量。
- 待确认导入会话数量。
- 最近七天的复习完成量与本周目标进度。

第一阶段可用现有列表接口在前端计算前两项，并将学习轨迹在缺少历史数据时显示为空状态。第二阶段应新增只读的工作台聚合接口，避免首页发起多次独立请求，也避免前端从完整列表推导历史趋势。

## 6. 响应式规则

- 宽度大于等于 1024px：固定侧栏，工作台双列展示学习轨迹与知识库概览。
- 宽度 768–1023px：侧栏收窄为图标导航，主内容保持单列优先。
- 宽度小于 768px：导航切换为顶部或底部栏，今日行动横幅、学习轨迹、知识库概览和快捷操作按单列排列。

## 7. 验收与测试

1. 首次进入应用默认显示工作台，且「开始复习」在有待复习卡片时位于首屏。
2. 工作台可正确显示无待复习、无知识、无历史趋势和有待确认导入等状态。
3. 从全局搜索进入搜索页时，关键词被正确带入并执行现有搜索逻辑。
4. 完成最后一张复习卡片后，工作台状态能反映完成结果。
5. 在桌面、平板和手机断点下，导航和首页模块无重叠、无横向溢出。
6. 现有导入、确认入库、知识库、复习和搜索页面测试继续通过，并为工作台新增组件测试。
